# Social realist phone-resistance spike

## Question

Can a tiny 3D-life-sim-shaped interaction communicate a Lifelines phone-practice attempt through visible resistance, object state, barks, and case interpretation — without collapsing into deterministic text-card outcomes?

## What this validates

Given a player assigns one Citizen Sleeper-style die to `Practice phone with Frank`, when the attempt resolves, then the player should be able to read whether the result came from phone fear, dignity defense, Frank proximity, Grete substitution risk, or low overskudd through the scene state and short casefile note.

## Scope

This is deliberately a self-contained HTML spike, not production Godot code.

It models:

- one apartment room as a spatial stage
- Elling, Frank, phone, chair, bedroom door, and script object
- dice-biased probability rather than deterministic face tables
- resistance stages: premise defense → annoyed compliance → pomp spiral → anger → retreat
- Ambjørnsen-ish bark text as systemic feedback
- persistent object/room state after each attempt
- a compact activity/skill lens for phone contact

## Non-goals

- no real 3D pathfinding
- no final UI styling
- no real content pipeline
- no production economy integration
- no deterministic dice branch table

## Test question

After five attempts with different dice and Frank proximity settings, does the player feel like they are watching Elling resist in a specific Norwegian/social-realist way, or just reading randomized flavor text?

## Verdict

Not run yet. Use the prototype and judge whether the resistance ladder and post-attempt state make the decision legible enough to justify a Godot spike.
