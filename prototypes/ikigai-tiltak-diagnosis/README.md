# Goal Probe / Tiltak Path spike

Throwaway HTML prototype for testing a revised Lifelines Tycoon loop.

## Design question

Is it more interesting to spend scarce state attention dice on uncovering a client-owned goal and then building a path toward it, instead of sorting observations into Ikigai quadrants?

The player starts with an unhealthy equilibrium and an unclear phrase from Elling: “something normal.” The work is to make that phrase actionable.

## Core loop tested

1. Roll state attention dice.
2. Spend a die on a probe, tiltak, or admin action.
3. Die quality changes the quality of the case signal, not just success/failure.
4. Goal hypothesis, blockers, path steps, and equilibrium bars update.
5. The player decides whether to gather more information, train a missing basic skill, create local usefulness, or attempt admin progression.

## Promising if

- The player has a concrete goal hypothesis to work toward.
- Low dice still teach something about the case.
- The progress bars feel useful as coarse readiness dimensions.
- Premature normal-job actions fail for legible reasons.
- The café-like goal becomes specific: ordered preparation, predictable scripts, low-risk public usefulness.

## Dead if

- The player feels they are clicking actions in a fixed optimal order.
- The goal hypothesis is too clean too early.
- The bars imply a full personality simulation that the design cannot support.
- Admin actions feel like progress without demonstrated readiness.
