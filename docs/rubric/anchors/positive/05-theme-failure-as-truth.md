# SDD-080 — failure pays out more knowledge than success

## Anchor

Paraphrased from the sibling project's SDD-080 (locked discipline #2, "trust × dice × knowledge"):

> Knowledge per topic has diminishing returns. A successful Tiltak confirms what was probably true. A failed Tiltak — the visit Elling didn't answer, the conversation he changed the subject from — surfaces a *new* part of the model. Failure pays MORE knowledge than success on the same topic.

A model trace:

- Day 4: Frank attempts `int_routine_visit`. Door not opened. 1.5h burned. Case file +1: `trauma:strangers_severe` (new tag, not previously observed).
- Day 7: Frank attempts `int_routine_visit` again, this time after sending a postcard the prior day. Door opened. Case file +1: `routine_tolerated_after_signal` (confirmation of an existing tag).

The day-4 entry is the larger knowledge gain. The mechanic enforces "failure pays more."

## Why this scores high on thematic-coherence

The bureaucratic frame insists that "failure" is not a player-grade. It is a citizen-truth: Elling didn't open the door because Elling has *this* relationship to unannounced visits. The mechanic IS the theme — when the system pays more for failure, the player learns to value the closed door, not avoid it. Sub-criterion 3 (failure = client truth) hits ceiling because failure is structurally non-punitive. Sub-criterion 4 (no RPG progression) hits because the knowledge gain is *about Elling*, not about the player's skill rating.

## Specific sub-criteria signal

- Sub-criterion 3 (Failure = client truth, not player error): 3 — failure produces a tag about Elling, costs nothing the player can't recover (overskudd regenerates), and is framed as discovery.
- Sub-criterion 4 (No RPG progression frame): 3 — the diminishing-returns curve attaches to *topics* (what we know about Elling), not to *the player* (what level the player has reached).
